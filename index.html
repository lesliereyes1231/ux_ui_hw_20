Fish and Wildlife Goverment Website Redesign

  • Research
  • Define
  • Ideate
  • Prototype
  • Test
  • Conclude

Problem

Fish and Wildlife website was in need of a redesign and better navigation and through user testing navigation was in disorder.

Solution

To redesign the website to help the flow for users by making the navigation easier and more efficient.

Details

Project Role: User Experience Designer and Researcher
Tools: Adobe Creative Cloud (Illustrator, Photoshop), Miro, Figma
Timeline: three weeks

Research

This projects user research consisted of conducted usability test on their current website so we can focus on and pin userâ€™s pain points. We conducted 5 usability test on the website to test their navigation system and overall lay out of website.

Task I asked users to complete were as follows: Apply for a single day volunteering opportunity Apply for a special use permit Buy a duck stamp Search for an endangered species. Some pain points we came across were Problems navigating and finding certain opportunities Overstimulated by amount of text and images Complicated navigation bars Catergories could be refined and tidied up more Some research/filter options were a bit confusing

Problem Statement

Fish and Wildlife's ran into the issue of a disorganized navigation system along with a lot of images and information that proved to be overwhelming.

Ideate

Using our insights from user testing, we did some open card sorting and reworked the website's information architecture since the biggest pain point for our users was navigation. It was hard for them to pin point specific topics

In this phase, we also developed our style guide for use in our later prototypes.

Prototype

We started designing how we could improve the navigation and making changes to the layout of the website for a more seamless channel of information

We created our mid-fidelity prototype by implementing our style guide doing some slight iteractions of icon choice & placement.

We wanted to ensure an intuitive navigation experience for our users, reconstructing our navigation was difficult as we wanted to keep all user flows in consideration but we had some conflicting data when it came to the infomation architecture

Testing & Final iterations

We iterated our high-fidelity prototype based on user feedback from testing our mid-fidelity prototype. Along with the desktop version we wanted to redesign the mobile to ensure no matter where the user accesses the website, it's as intuitive as the desktop

Conclusions

This redesign was a great learning experience, one thing we can take away from this project is centering this around the users needs and familiarizing ourselves with figma micro animations a bit more. If we were to continue elaborating on this redesign we would create more micro-animations and use auto layout more for smoother more even animations

Overall, it was a good opportunity to learn how to design and prototype for desktop and mobile and keeping a consistent but efficient look and feel to both devices.
